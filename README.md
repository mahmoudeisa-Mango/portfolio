# Case Files — Mahmoud Eisa

A portfolio site showcasing research and analysis work, built as a static site for GitHub Pages.

## Structure
```
index.html              # the whole site (single file, no build step)
reports/
  tmg-financial-ratio-analysis.pdf
  aastmt-student-satisfaction-study.pdf
```

## Deploying to GitHub Pages
1. Create a new repo (e.g. `mahmoudeisa-Mango.github.io` for a root domain, or any name like `portfolio`).
2. Upload `index.html` and the `reports/` folder to the repo, keeping this same folder structure.
3. In the repo, go to **Settings → Pages**.
4. Under **Source**, select the `main` branch and `/ (root)` folder, then save.
5. GitHub will give you a live URL within a minute or two (e.g. `https://mahmoudeisa-Mango.github.io/portfolio/` or `https://mahmoudeisa-Mango.github.io/` if you named the repo `mahmoudeisa-Mango.github.io`).
